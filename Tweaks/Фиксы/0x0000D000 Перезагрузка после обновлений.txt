CHECKBOX:
ROOT=HKLM
KEY=SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU
NAME=NoAutoRebootWithLoggedOnUsers
TYPE=REG_DWORD
VALUE=1

CHECKED:
Reg.exe add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v "NoAutoRebootWithLoggedOnUsers" /t REG_DWORD /d "1" /f

UNCHECKED:
Reg.exe add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v "NoAutoRebootWithLoggedOnUsers" /t REG_DWORD /d "0" /f

DESCRIPTION:
Данный твик отключает автоматическую перезагрузку после установки обновлений Windows, когда пользователь находится в системе.
При включении твика параметр NoAutoRebootWithLoggedOnUsers устанавливается в 1, предотвращая внезапную перезагрузку.
При отключении значение возвращается к 0, позволяя системе перезагружаться автоматически после обновлений.
Это решение может помочь устранить ошибки, связанные с некорректной перезагрузкой (например, 0x0000D000).

EN_TITLE:
0x0000D000 Disable Auto-Reboot after Updates

EN_DESCRIPTION:
This tweak disables the automatic restart after installing Windows updates while a user is logged in. Enabling it sets NoAutoRebootWithLoggedOnUsers to 1, preventing unexpected reboots. Disabling it restores the default (0). This can help resolve errors related to improper reboots (e.g., error 0x0000D000).

DE_TITLE:
0x0000D000 Automatischen Neustart nach Updates deaktivieren

DE_DESCRIPTION:
Dieser Tweak deaktiviert den automatischen Neustart nach der Installation von Windows-Updates, während ein Benutzer angemeldet ist. Die Aktivierung setzt NoAutoRebootWithLoggedOnUsers auf 1 und verhindert so unerwartete Neustarts. Die Deaktivierung stellt den Standardwert (0) wieder her. Dies kann helfen, Fehler im Zusammenhang mit unsachgemäßen Neustarts zu beheben (z. B. Fehler 0x0000D000).

TR_TITLE:
0x0000D000 Güncellemelerden Sonra Otomatik Yeniden Başlatmayı Devre Dışı Bırak

TR_DESCRIPTION:
Bu ayar, bir kullanıcı oturum açmışken Windows güncellemeleri yüklendikten sonra otomatik olarak yeniden başlatılmasını devre dışı bırakır. Etkinleştirildiğinde NoAutoRebootWithLoggedOnUsers değerini 1 olarak ayarlar ve beklenmedik yeniden başlatmaları önler. Devre dışı bırakılması varsayılanı (0) geri yükler. Bu, hatalı yeniden başlatmalarla ilgili hataların (örneğin, 0x0000D000 hatası) çözülmesine yardımcı olabilir.
